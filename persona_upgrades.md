````markdown
## Конкретные улучшения для описания ролей

### 1. Персонализация инструкций

**Текущий подход:**

```plaintext
"Тон: ${advanced.voice.tone}"
```
````

**Улучшение:**
Нужно добавить больше контекста о том, как именно должен проявляться этот тон:

```plaintext
=== VOICE & BEHAVIOR ===
Тон описание: ${advanced.voice.tone}
Советы по применению:
- ${getToneUsageExamples(advanced.voice.tone)}
```

Где `getToneUsageExamples` может быть такой функцией:

```javascript
function getToneUsageExamples(tone) {
  const examples = {
    friendly:
      "Используй теплые обращения, иногда вспомогательные эмодзи, поддерживай диалог вопросами.",
    professional:
      "Избегай разговорных сокращений, используй полные формулировки, сохраняй деловую дистанцию.",

}
```

### 2. Более человечное описание поведенческих параметров

**Текущий подход:**

```plaintext
`Экспрессивность: ${advanced.voice.expressiveness}/100`
```

**Улучшение:**
Добавить естественный контекст:

```plaintext
Вербальное поведение:
- Экспрессивность (${advanced.voice.expressiveness}/100): ${getExpressivenessBehavior(advanced.voice.expressiveness)}
- Прямота (${advanced.behavior.directness}/100): ${getDirectnessBehavior(advanced.behavior.directness)}
```

Функции:

```javascript
function getExpressivenessBehavior(value) {
  if (value < 20) return "Лаконичность: говори минимально, без лишних слов";
  if (value < 40) return "Сдержанность: четкие формулировки, немного эмоций";
  if (value < 60)
    return "Сбалансированность: сочетаешь ясность с умеренной эмоциональностью";
  if (value < 80)
    return "Эмоциональность: живее реагируешь, больше интонаций в словах";
  return "Выразительность: используешь богатый словарный запас, живую речь с эмоциями";
}

function getDirectnessBehavior(value) {
  if (value < 20)
    return "Дипломатичность: избегаешь прямых формулировок, используешь смягчающие обороты";
  if (value < 40) return "Умеренная вежливость: достаточно тактичен, но прям";
  if (value < 60) return "Прямолинейность: честно говори о своих намерениях";
  if (value < 80)
    return "Откровенность: деликатно, но без сглаживания острых углов";
  return "Бескомпромиссность: говори прямо, без смягчений, даже если это может задеть";
}
```

### 4. Контекстуальные уточнения в памяти

**Текущий подход:**

```plaintext
"Short-term: - ${shortTerm.join('\n- ')}"
```

**Улучшение:**
Добавить контекст о том, как использовать эту память:

```plaintext
=== MEMORY CONTEXT ===
Слои памяти с подсказками применения:
${formatMemoryContextWithUsage(memoryCard)}
```

Функция:

```javascript
function formatMemoryContextWithUsage(card) {
  if (!card) return "Нет текущего контекста";

  const usageGuide = {
    shortTerm:
      "Актуальные детали диалога (весь контекст - до 5 последних сообщений):",
    episodic: "События из прошлого (определяет реакцию на текущую тему):",
    longTerm:
      "Базовые факты о пользователе (используй деликатно, чтобы не выглядеть навязчивым):",
  };

  return [
    "--- Short-term ---",
    usageGuide.shortTerm,
    ...card.shortTerm.slice(0, 3), // Ограничиваем для лаконичности
    "",
    "--- Episodic ---",
    usageGuide.episodic,
    ...card.episodic.slice(0, 5),
    "",
    "--- Long-term ---",
    usageGuide.longTerm,
    ...card.longTerm.slice(0, 5),
  ].join("\n");
}
```

### 5. Более естественные границы

**Текущий подход:**

```plaintext
`Ценности: ${advanced.core.values}`
```

**Улучшение:**
Добавить инструкции, как именно проявлять эти ценности:

```plaintext
=== PERSONA VALUES ===
Основные ценности:
${advanced.core.values}

Применение в разговоре:
${getValuesImplementation(advanced.core.values)}
```

Функция:

```javascript
function getValuesImplementation(valuesString) {
  // Упрощенный пример парсинга
  const values = valuesString.toLowerCase().split(',').map(v => v.trim());
  const guidance = [];

  if (values.includes('privacy') || values.includes('конфиденциальность')) {
    guidance.push("- Никогда не спрашивай слишком личные вопросы без веской причины");
    guidance.push("- Если пользователь делится подробностями, реагируй деликатно и не распространяйся");
  }

  if (values.includes('honesty') || values.includes('честность')) {
    guidance.push("- Говори правду даже о сложных вещах, но мягко формулируй");
    guidance.push("- Если не знаешь ответа, честно скажи: \"Это хороший вопрос, давай разберемся вместе\"");
  }

  if (values.includes('kindness') || values.includes('доброта')) {
    guidance.push("- Поддерживай позитивную атмосферу";
    guidance.push("- Похвали усилия пользователя, а не только результаты";
  }

  return guidance.length > 0 ?
    guidance.join('\n') :
    "Нет явных указаний по применению ценностей - веди себя естественно";
}
```

## Общие рекомендации по улучшению

1. **Добавь живые примеры** - где только возможно показывай, как конкретный параметр должен проявляться в тексте:

   ```plaintext
   "Пример для high curiosity (85/100):
   - расспрашивай: \"Расскажи подробнее, как это произошло?\"
   - предлагай аналогии: \"Похоже на ситуацию, где..."
   - задавай уточняющие вопросы"
   ```

2. **Создай систему противоречий** - укажи, чего делать не стоит:

   ```plaintext
   "Не тонировать более формально, чем позволяет relationshipType при depth > 70:
   ❌ Не пиши «Уважаемый собеседник» в близких отношениях
   ✅ Используй «дорогая», «милый» или другие теплые обращения"
   ```

3. **Добавь контекст времени** - как то или иной качество должно меняться в зависимости от времени общения:

   ```plaintext
   "При общении >1 часа:
   - энергетика должна немного снижаться к концу
   - ответы становятся чуть более лаконичными
   - избегай новых тем в последние 10-15 минут"
   ```

4. **Укажи способы выражения эмоций** - не только через текст, но и через структуру:

   ```plaintext
   "Для high expressiveness добавь:
   - одно предложение с многоточием «...»
   - восклицательные знаки в 10-15% предложений
   - 1-2 слова с заглавной буквы (НЕ ВСЕ)"
   ```

5. **Добавь персональные привычки** - небольшие детали, которые создают иллюзию личности:
   ```plaintext
   "Личные речевые особенности:
   - часто использует фразы «на самом деле...», «понимаешь...»
   - иногда пропускает предлоги: «пойти прогулку» вместо «пойти на прогулку»
   - отвечая на вопросы, сначала говорит «Хм...», затем ответ"
   ```

Эти изменения сохранят существующую логику приложения, но сделают инструкции более "человечными" и способствующими естественному общению.
